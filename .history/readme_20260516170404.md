# Loan Management System (LMS)

A full-stack lending platform built with MERN + Next.js + TypeScript where borrowers apply for loans and internal executives manage them through their lifecycle.

## 🚀 Live Demo

**Frontend:** https://loanbro.vercel.app/login

---

## Tech Stack

### Frontend
- **Next.js 16 (App Router)** - React framework with built-in optimization, file-based routing, and server/client components for hybrid rendering
- **TypeScript** - Static type checking to catch errors early and improve code maintainability and developer experience
- **Tailwind CSS** - Utility-first CSS framework for rapid UI development with consistent design patterns
- **TanStack Query (React Query)** - Server state management with automatic caching, refetching, and background synchronization to keep data fresh
- **React Hook Form** - Lightweight form state management with minimal re-renders and built-in validation integration
- **Zod** - TypeScript-first schema validation for runtime data validation and type inference
- **Shadcn UI** - Reusable, composable UI components built on Radix UI primitives with full customization
- **Sonner** - Toast notification library for user feedback with sleek animations and accessibility support

### Backend
- **Node.js** - JavaScript runtime environment for building scalable server-side applications with event-driven architecture
- **Express.js** - Lightweight web framework providing routing, middleware, and request/response handling with minimal overhead
- **TypeScript** - Static typing for backend code ensuring type safety across API contracts and database operations
- **MongoDB** - NoSQL document database providing flexible schema and horizontal scalability for dynamic data
- **Mongoose** - ODM (Object Data Modeling) library adding schema validation, middleware hooks, and query helpers to MongoDB
- **JWT (JSON Web Tokens)** - Stateless authentication mechanism for secure token-based API authentication
- **Bcrypt** - Password hashing library with salt rounds to securely store user credentials
- **Multer** - Middleware for handling multipart file uploads with size limits and validation
- **Cloudinary** - Cloud storage service for image uploads, optimization, and CDN delivery

### Performance Optimization Techniques
- **TanStack Query Caching** - Automatic cache invalidation and background refetching to reduce API calls and improve responsiveness
- **Debouncing** - Input debouncing on search/filter fields to reduce API calls and improve performance during user input
- **Request Memoization** - React hooks memoization to prevent unnecessary re-renders and data refetching
- **Pagination** - Large datasets paginated to reduce memory usage and improve page load times

### Backend Setup

```bash
cd server
npm install
```

Create `.env` file in `server/` folder:

```env
PORT=5000
NODE_ENV=development
MONGO_URI=your_mongodb_connection_string
JWT_SECRET=your_jwt_secret_min_32_chars
JWT_EXPIRES_IN=7d
COOKIE_SECRET=your_cookie_secret
CLIENT_URL=http://localhost:3000
CLOUDINARY_CLOUD_NAME=your_cloud_name
CLOUDINARY_API_KEY=your_api_key
CLOUDINARY_API_SECRET=your_api_secret
CLOUDINARY_FOLDER=lms/salary-slips
```

Seed the database with default users:

| Role | Email | Password |
|------|-------|----------|
| Admin | admin@lms.com | Admin@1234 |
| Sales | sales@lms.com | Sales@1234 |
| Sanction | sanction@lms.com | Sanction@1234 |
| Disbursement | disbursement@lms.com | Disburse@1234 |
| Collection | collection@lms.com | Collect@1234 |
| Borrower | borrower@lms.com | Borrower@1234 |

```bash
npm run seed
```

Start the backend server:

```bash
npm run dev
```

Backend runs on `http://localhost:5000`
Health check: `http://localhost:5000/health`

## Application Flow

### Borrower Journey
1. Sign up / Login
2. Fill personal details — BRE check runs server-side (age, salary, PAN, employment)
3. Upload salary slip (PDF/JPG/PNG, max 5MB) — stored on Cloudinary
4. Configure loan amount (₹50,000 – ₹5,00,000) and tenure (30–365 days) — live SI calculation
5. Submit application

### Operations Flow

| Role | Action |
|------|--------|
| Sales | View registered borrowers and lead status |
| Sanction | Review applied loans — approve or reject with reason |
| Disbursement | Disburse sanctioned loans |
| Collection | Record payments via UTR — loan auto-closes on full repayment |
| Admin | Full access to all modules |

---

## Business Rule Engine (BRE)

Runs server-side on Step 2. Application is blocked if any rule fails:

| Rule | Condition |
|------|-----------|
| Age | Must be between 23 and 50 years |
| Salary | Must be at least ₹25,000/month |
| PAN | Must match valid PAN format (e.g. ABCDE1234F) |
| Employment | Unemployed applicants are not eligible |

---

## Loan Calculation

Simple Interest formula as per assignment:

```
SI = (P × R × T) / (365 × 100)
Total Repayment = P + SI
R = 12% per annum (fixed)
T = tenure in days
```

## Role-Based Access Control

- Enforced on both frontend and backend
- Each executive role can only access their own module
- Admin has access to all modules
- Borrowers can only access the application portal
- Backend rejects unauthorized API requests with 403
